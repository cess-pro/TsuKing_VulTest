# 判定标准

## RD

发送不带有RD标识的请求，在权威侧判断是否收到该查询

## 负缓存

- 方案一：基础判断是否对ServerFailure、timeout进行缓存；

    能有缓解，不能阻断攻击

- 方案二：判断是否对“已经失效的权威”进行缓存，即对于不同的恶意“子域名”均不再查询；

    能够阻断攻击

代码实现在于Prob文件中，负缓存测试时，是否请求不同的子域名。

判断标准是连续请求的耗时的**中位数**，或许可以换其他方式
